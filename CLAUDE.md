# Circuit Defense

A browser tower defense game. **The game is `tower-defense.html` (markup) + `tower-defense.css` (styles) + seven domain-split JS files (v1.8.2).** Everything is wired with classic `<link rel="stylesheet">` and `<script src>` tags ‚Äî **NEVER ES modules** (`type="module"` breaks `file://`). No build step, no dependencies, no assets, no network: double-click `tower-defense.html` to play offline. All graphics are canvas-drawn, all sound is synthesized via WebAudio (oscillators + filtered white noise).

**JS load order (classic scripts share one global scope, so order = dependency order ‚Äî do not reorder, and keep the startup-init block last in `cd-render.js`):**

1. `cd-core.js` ‚Äî canvas refs (`cv`/`ctx`/`W`/`H`), `GAME_VERSION`/`CHANGELOG_ENTRIES` + What's-New panel, audio (`tone`/`noise`/`SFX`)
2. `cd-maps.js` ‚Äî `MAPS`/`genPathWith`/`buildPath`/`pointAt`, mayhem `WAVE_MODS`, stars, difficulty
3. `cd-defs.js` ‚Äî `TALENTS`/`loadMeta`/meta, `TOWER_TYPES`/`MODES`/`SPECS`, `ABILITIES`, `PERKS`/`REPEATABLE`/`openDraft`
4. `cd-state.js` ‚Äî run state globals, kill-combo vars, `saveRun`/`loadRun`/`resetState`
5. `cd-game.js` ‚Äî start screen/`beginGame`, enemies/`startWave`, shop, upgrade panel, `eff*` stats, input listeners
6. `cd-update.js` ‚Äî `update()`, `ACHIEVEMENTS`/`grantAchievements`, `recordBest`/`applyRecordFlourish`, `endGame`/`winGame`
7. `cd-render.js` ‚Äî `draw()`, the rAF `loop`, and the startup init calls (`loadMeta`/`buildPath`/`renderStartScreen`/`resetState`/`setActiveUI`/`initWhatsNew`/`requestAnimationFrame(loop)`)

When adding code, drop it in the file whose domain it matches; new globals are visible everywhere as long as the file defining them loads before any top-level code that runs them (function bodies are fine anywhere ‚Äî they resolve at call time).

## Running / testing

- Dev server: `.claude/launch.json` defines a `game` config (`python -m http.server 8123`). Use the preview tools and navigate to `/tower-defense.html`. (Note: `preview_screenshot` tends to time out because the rAF render loop keeps the page busy ‚Äî verify state with `preview_eval`/`preview_snapshot` instead.)
- **Automated test harness lives in `tests/`** (Node + Playwright, dev-only ‚Äî the shipped game is just the three raw files, no build step). Run with `cd tests && npm install && npx playwright install chromium && npm test`; exit code 0 = green. It drives the real `tower-defense.html` headlessly via an injected in-page driver: `__cdGodTowers(n)` pushes near-invincible towers, `__cdDrive({maxWave, cap})` runs the sim with `update(1/60)` to a wave boundary or game end, auto-picking drafts. Use the `check(...)` helper for assertions; add one whenever you add behavior. Before committing, spawn a subagent to run `npm test`.
- **The game loop runs on `requestAnimationFrame`, so it pauses when the tab is hidden.** Headless testing through preview_eval must drive the simulation manually: call `update(1/60)` in a loop instead of waiting wall-clock time.
- Standard test recipe (via preview_eval):
  1. `beginGame()` (set `gameMode`/`mapKey`/`diffKey` first), give `gold`, push tower objects directly into `towers`
  2. `startWave()`, then `while (!done) update(1/60)`
  3. If `draftOpen`, click a card: `document.getElementById('draftCards').children[0].click()`
  4. Check `preview_console_logs` for errors afterwards
- Synthetic canvas clicks must set real `clientX/clientY` relative to `cv.getBoundingClientRect()` ‚Äî the click handler recomputes coords from the event.
- **Always clean up test data afterwards**: remove localStorage keys `cd_save`, `cd_meta`, `cd_campaign`, `cd_best_easy/normal/hard`, reset `meta = {chips:0, talents:{}}; loadMeta()`, and call `backToMenu()` so the user starts fresh.

## Architecture (code split across the seven `cd-*.js` files above)

Rough section order in the file:

- **Audio**: `tone()` (oscillator), `noise()` (filtered white noise buffer), `SFX` object ‚Äî every tower type has a distinct sound (tesla = lightning crackle, cannon = sub-bass boom, etc.)
- **Maps & paths**: `MAPS` (classic/spiral/serpent/mayhem), `genPathWith(rnd)` generates random axis-aligned serpentine paths, `buildPath()` fills `waypoints/segs/pathLen`, `pointAt(dist)`, `distToPath(x,y)`
- **Modes**: `gameMode` = `'quick'` | `'campaign'`; `campLevel` 1‚Äì40, `CAMPAIGN_LEVELS = 40`. Campaign maps are **freshly random each attempt** (deliberate design choice). `victoryWave()` = `14 + campLevel` for campaign (L1=15 ‚Ä¶ L40=54), 30 for quick. Campaign enemy HP scales `1 + (campLevel-1)*0.04`. Base per-enemy HP (`enemyTemplate`) = `(18 + w*7 + w^1.9) * 1.80 * d.hp * campScale` ‚Äî the global multiplier is raised iteratively per recurring "too easy" FEEDBACK: `1.2` ‚Üí `1.44` (v1.9.2, +20%) ‚Üí `1.80` (v1.10.0, +25% more). Uniform so the swing is exactly +25% at every wave incl. deep endless (each step within the ‚â§25%/number/run guardrail); bosses/tanks scale off it too. Progress = localStorage `cd_campaign` (highest completed level). Mayhem (quick-mode map) shifts the path + relocates towers every 5 waves and rolls per-wave `WAVE_MODS`.
- **Meta / talents**: `TALENTS` (20 talents in CORE + TOWER MASTERY sections, ~2290 chips to max), `meta = {chips, talents, achievements, stats}` in localStorage `cd_meta`. Chips earned per run via `chipsForRun()`. `maxTowerLevel()` = 5 + overdrive rank. `loadMeta()` additively migrates older saves (defaults `achievements`/`stats` when absent ‚Äî never break this).
- **Achievements** (v1.5.0, 9 badges as of v1.8.0): `ACHIEVEMENTS` array + `ACH_BY_ID`. `meta.achievements` is an id‚Üítrue map; `meta.stats = {dmg, runs, bestCombo}` are lifetime counters (`bestCombo` added v1.8.0, defaulted to 0 in `loadMeta()`). `grantAchievements(won)` is called once per finished run from `endGame(false)`/`winGame(true)` ‚Äî it tallies `stats` (including `meta.stats.bestCombo = max(bestCombo, comboBest)` from the run's peak streak), evaluates each badge's condition, and returns the newly-unlocked list, which `achLine()` appends to the overlay text. Per-run `livesLostThisRun` flag (set wherever a life leaks; forced `true` in `loadRun()` so resumed runs can't earn **Flawless**). The **Combo Master** badge (`combo30`, v1.8.0) is granted when the run's `comboBest >= 30`. Start-screen üèÖ button ‚Üí `openAchievements()`/`renderAchievements()` panel (`#achPanel`, mirrors talents); `#achBtn` shows `done/total` (updated in `renderStartScreen`). Adding a badge = push to `ACHIEVEMENTS` + a condition line in `grantAchievements`.
- **Records / personal bests** (v1.6.0): start-screen üèÜ button ‚Üí `openBests()`/`renderBests()` panel (`#bestPanel`, mirrors achievements). Shows a `.bestTbl` matrix of highest wave per quick-mode **map √ó difficulty**, an `‚òÖ Any map` row (from the legacy per-diff bests), and a `.bestStats` footer (campaign level, lifetime dmg, **best combo** [v1.8.0], runs, chips). Best-wave is recorded by `recordBest()` (called from `endGame`/`winGame`, replacing the old inline `if (wave > best)` lines): it updates the legacy `cd_best_<diff>` key (via `bestKey()`) **and**, in quick mode only, a new additive `cd_best_<map>_<diff>` key. Campaign runs never write per-map keys (maps are random per attempt). All keys read with `|| 0` fallback ‚Äî old saves unaffected. Map keys (classic/spiral/serpent/mayhem) can't collide with diff keys (easy/normal/hard). **New-record flourish (v1.6.1):** `recordBest()` returns a `{prev, now}` event when a quick-mode run beats its per-map cell (only when `prev > 0` ‚Äî first-ever entries record silently and return `null`); `endGame()`/`winGame()` pass it to `applyRecordFlourish(rec)`, which shows the golden `#ovRecord` banner (toggled by an `.record` class on `#overlay`, CSS pop/pulse animations), plays `SFX.record()` (delayed ~450ms), and adds shake + a particle burst. `applyRecordFlourish(null)` clears the banner, so every overlay show resets it (no stale banner). DOM/audio only ‚Äî no save impact; campaign never fires it.
- **Version / What's New**: `GAME_VERSION` constant (semver-ish, bumped every routine run) + `CHANGELOG_ENTRIES` array (last ~10, newest first, mirrors CHANGELOG.md) near the top of the script. **Layout (as of v1.4.1):** the whole game lives in a `#gameCol` column (h1, `#hud`, `#stage`‚Üí`#gameWrap`/canvas, `#shop`, `#controls`, `.hint`), and the `#whatsnew` `<aside>` is its **sibling** inside an outer `#appRow` flex row (`align-items:flex-start; flex-wrap:wrap`). So the panel floats beside the **entire game** ‚Äî opening it shifts the whole column together (not just the canvas, which was the v1.4.0 complaint). The panel is **open by default**, top-aligned with the game column, flush against its right edge (`#appRow` gap 0, `border-left:none`, `border-radius:0 8px 8px 0`, `align-self:stretch`), **height-capped to the game** (v1.5.2): a `syncWhatsNewHeight()` helper sets the panel's inline `max-height` to `#gameCol.offsetHeight` (called from `openWhatsNew()` and on `window` `resize`), so on tall viewports it never overhangs the bottom of the board ‚Äî overflowing entries scroll internally via `#wnList` instead. (The static CSS `max-height:88vh` stays as a no-JS fallback but the inline cap wins; the row is `align-items:flex-start`, so `#gameCol`'s height is independent of the panel and there's no feedback loop.) It wraps below `#gameCol` on narrow viewports. `initWhatsNew()` (called at startup) opens it unless the player closed it before ‚Äî `closeWhatsNew()` persists `cd_wnclosed='1'`, `openWhatsNew()` clears it. The start screen has a `‚ú® What's New` button + clickable `#verTag` (version, set in `renderStartScreen`) to reopen, and the panel has a `‚úï` (`#wnClose`) to hide it. When you ship a run, bump `GAME_VERSION` and prepend a `CHANGELOG_ENTRIES` entry.
- **Towers**: `TOWER_TYPES` (gun/sniper/frost/cannon/tesla/poison/buff), `SPECS` (2 specializations per type, chosen free at level 5+), targeting `MODES` (first/last/strong/close). **Spec rework (v1.10.0, owner FEEDBACK "dominated specs"):** to keep each L5 choice a real axis (not a trap where one option has zero damage upside), booster **Network** now adds **+10% aura power** on top of +50% range (wired in `effBuffPower`), and cannon **Mega Blast** now adds **+15% damage** alongside its blast-radius boost (`effDmg` + the `1.6` radius mult in `hitEnemy`). Overclock (+20% power) / Cluster (+50% dmg) remain the concentrated-power picks. Frost **Shatter** was *nerfed* the same run (`√ó6 ‚Üí √ó4.5`, ‚àí25%): paired with Frost Mastery + a L6 booster it was a run-warping solo carry (owner mid-run sim: frost doing 764 dmg). The booster aura scaling (`buffPower` +0.1/level ‚Üí +75% at L6) and Frost Mastery are noted on ROADMAP as the next snowball levers.
- **Poison** (`proj:'poison'`, base dmg **7** as of v1.10.0): on hit, `hitEnemy()` applies a DoT `{dps: p.dmg*2.6, t: perkState.poisonDur}` (Virulent spec √ó2 dps, Lingering √ó2 dur, Virology perk +2s) **and corrodes the target's armor by ‚àí3 (floored at 0)** ‚Äî a hard counter to shield/armored/boss enemies (owner FEEDBACK "poison should reduce enemy defense + more DoT", v1.10.0). DoT ticks via `damage(e, dps*dt, src, silent)` each frame (per-tick dmg is tiny so armor barely blunts it; the up-front impact hit is what armor corrosion helps). All run-only/live-computed ‚Äî no save impact.
- **Abilities**: `ABILITIES` (meteor Q / freeze W / gold rush E), `abilityCd`, meteor is click-to-target via `armedAbility`
- **Run perks**: `PERKS` with rarity tiers (common/rare/legendary), drafted 1-of-3 every 5 waves (`openDraft`, weighted by `rollRarity()` ‚âà 78/14/8 common/rare/legendary). Legendary = "super grades" (Diamond Core, Midas, Orbital Support, Ascension‚Ä¶). `perkState` (from `freshPerkState()`) holds all run-modifier numbers; `REPEATABLE` perks can be offered again.
- **Effective stats**: `effDmg/effRate/effRange/effBuffPower/effBuffRange` combine base stats √ó talents √ó perks √ó specs √ó buffs √ó wave mods. **Booster auras do NOT stack ‚Äî `buffMultFor` takes the max** (deliberate; UI says so).
- **Save/resume**: `saveRun()` after each wave + on quit (mid-wave quit saves `wave-1`); `loadRun()` rebuilds tower stats from base √ó level multipliers (dmg √ó1.45, range √ó1.08, rate √ó0.88 per level). Random-path modes (mayhem/campaign) call `relocateTowers()` on resume.
- **Game flow**: `startWave ‚Üí endWave` (interest, draft every 5 waves, auto-start timer) ‚Üí `winGame`/`endGame` (chips awarded, MVP/perk lines) ‚Üí `continueEndless`/`nextLevel`/`backToMenu`. `quitRun()` needs two clicks within 3s.
- **Idle chrome (v1.5.1)**: when no game is active, the in-game chrome (stats `#hud`, tower `#shop`, wave `#controls`, hotkey `.hint`) is dimmed (opacity .3 + grayscale + `pointer-events:none`) via an `.idle` class on `#gameCol`. `setActiveUI()` toggles it from the `started` flag and is called in `beginGame`/`loadRun` (lit), `backToMenu` (dimmed), and at startup. Chrome stays lit during the game-over overlay (still an active session); re-dims only on return to menu. Pure CSS + one helper, no save impact.
- **Kill-streak combo (v1.7.0)**: consecutive kills within `COMBO_WINDOW` (2s) build a streak. Run-only state `comboCount/comboTimer/comboBest/comboFlash` (declared by the globals, **reset in `resetState()`**, never saved). `damage()` bumps `comboCount`/refreshes `comboTimer`/sets `comboFlash=1` on every death and, at milestones (`=== 5 || (>=10 && %10===0)`), fires `SFX.combo(n)` (rising chirp, pitch by tier) + shake + an `addExplosion` burst + a `üî• N√ó COMBO!` floater ‚Äî **both on the center board, just below the top HUD band** (`W/2,114` / `W/2,132`) as of v1.8.4, so they clear the bottom-right combo meter, the centered "Wave clear! +bonus" round-clear floater (~y36-90), and the centered boss HP bar (~y8-32). `update()` decays `comboTimer` (‚Üí resets `comboCount` to 0 at 0) and `comboFlash`. `draw()` renders the meter when `comboCount>=2 && comboTimer>0` in the **bottom-right corner** (`ax=W-16`, number baseline `H-26`) as of **v1.8.5** ‚Äî its own empty space, clear of the whole top band (boss bar, round-clear bonus, top-right `#abilityBar`), the centered milestone pop, and the bottom-**left** wave preview: the `N√ó` multiplier is right-aligned to the corner with the "COMBO" label to its **left** on the same baseline, and a draining timer bar in its **own lane below** (`y=H-16`, drains toward the corner). Right-anchored so wide counts (`100√ó`) grow leftward, not off-edge. (History: meter was top-right in v1.7.0 ‚Üí top-left v1.7.1 ‚Üí bar/label relayout v1.8.3 ‚Üí bottom-right v1.8.5; the v1.8.3 bar-under-label and the top-edge clipping at top-left both drove the corner move.) Color from `comboColor(n)` (green‚Üígold‚Üíorange‚Üíred‚Üípurple at 10/20/30/50), pulse-scaled by `comboFlash`. **Purely cosmetic in-game ‚Äî no gold/economy impact.** The combo state vars are run-only and never saved; the only persistent hook is at end-of-run, where `grantAchievements()` reads the run's `comboBest` peak into the lifetime `meta.stats.bestCombo` stat and grants the **Combo Master** badge at `>= 30` (v1.8.0).
- **Rendering**: pseudo-3D canvas ‚Äî shaded cylinder towers, radial-gradient sphere enemies, ground shadows, path depth bevel, projectile trails, starfield, vignette, boss HP bar, screen `shake`. **Reduced motion (v1.10.0):** `reduceMotion()` (`cd-core.js`, reads `matchMedia('(prefers-reduced-motion: reduce)')` live, guarded for no-`matchMedia` envs) gates the `shake` translate at the top of `draw()` ‚Äî OS "reduce motion" ‚Üí no screen shake. (Particle/explosion gating + an in-game toggle are noted ROADMAP follow-ups.) The combo timer bar also blinks red in its last third (near-miss cue, derived from `comboTimer` ‚Äî no wall clock).
- **Upgrade panel**: rebuilds itself when `upgradeKey(t)` changes (gold affordability, level, spec, *effective* stats) ‚Äî this is how it live-updates when boosters/perks land while open. Keep new stat sources reflected in `upgradeKey`. **Position (v1.9.1):** `showUpgrade()` pins the `#upgradePanel` to the **lower-left corner** (`left:10px`, bottom-anchored `top:auto; bottom:10px`) ‚Äî *not* at the clicked tower ‚Äî so the menu never covers live combat on the path (owner FEEDBACK). Bottom-anchored so the tall L5 spec-choice panel grows upward without clipping off the canvas bottom. (It shares the corner with the faint between-wave `Next:` preview, but that only shows while `!waveActive`.) Test group `[15]` asserts the corner pin + tower-independence.

## Conventions & gotchas

- localStorage keys all prefixed `cd_`: `cd_save`, `cd_meta`, `cd_campaign`, `cd_best_<diff>`, `cd_best_<map>_<diff>` (per-map quick-mode best, v1.6.0), `cd_mute`, `cd_speed` (1√ó/2√ó/3√ó game-speed pref, v1.10.0), `cd_wnclosed` (What's New panel hidden by player)
- **Document metadata** (v1.8.6, in `tower-defense.html` `<head>`): the favicon is an **inline SVG data URI** (gold ‚ö° bolt) ‚Äî keep it inline so it needs no network and works on double-click `file://`; never switch it to a separate `favicon.ico` file/URL. Head also carries `viewport`/`description`/`theme-color`/Open Graph tags. Test group `[13]` asserts these exist.
- Speed toggle just runs `update()` 1‚Äì3√ó per frame (the rAF `loop` in cd-render.js does `for (i<speed) update(dt)`); pause/draft/menu gate at the top of `update()`. **`speed` is persisted in `localStorage.cd_speed` (v1.10.0)** ‚Äî restored at declaration in cd-state.js, written in `toggleSpeed()`, button label synced at startup. Before this, every page load reset speed to 1√ó, so a refresh+Resume silently dropped to 1√ó and *all* towers fired at base cadence (owner "shoot at original speed" bug; the per-tower stat restore was always correct ‚Äî `effRate`/range of a resumed tower are byte-identical to a fresh one).
- `damage()` handles armor (reduced by piercing talent), kill credit (`src.dealt/kills`), bounty (fortune talent, midas perk, bountyAdd), splitter children via `pendingSpawns`
- Enemy kinds: norm, fast (w3+), tank (w5+), heal (w7+), shield/armored (w9+), split (w11+), **phantom (w13+)**, boss every 5th wave. **Phantom** (`#39d0d8`, slot `i%6===5`) blinks forward 58px every ~2s and sets `e.blinkInvuln=0.35s` ‚Äî while >0 it's skipped by `pickTarget()` and `damage()` returns early (intangible). `e.blinkCd` only ticks when `frozen<=0`, so freeze counters it. Render-translucent (cd-render.js) with a üëª glyph. Blink fields are run-only (enemies never saved mid-wave) ‚Äî no save migration.
- When adding a perk/talent that changes a stat, wire it into the corresponding `eff*()` helper AND `upgradeKey()` if it shows in the panel
- User's stated preferences so far: loves additive/addictive loops, wanted talents expensive/grindy (~50 runs to max), campaign maps random per attempt, booster non-stacking is fine but must be labeled
