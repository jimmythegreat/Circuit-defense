# Circuit Defense

A browser tower defense game. **Everything lives in one self-contained file: `tower-defense.html`** — no build step, no dependencies, no assets. Open it in any browser to play. All graphics are canvas-drawn, all sound is synthesized via WebAudio (oscillators + filtered white noise).

## Running / testing

- Dev server: `.claude/launch.json` defines a `game` config (`python -m http.server 8123`). Use the preview tools and navigate to `/tower-defense.html`. (Note: `preview_screenshot` tends to time out because the rAF render loop keeps the page busy — verify state with `preview_eval`/`preview_snapshot` instead.)
- **Automated test harness lives in `tests/`** (Node + Playwright, dev-only — the shipped game is still the single HTML file, no build step). Run with `cd tests && npm install && npx playwright install chromium && npm test`; exit code 0 = green. It drives the real `tower-defense.html` headlessly via an injected in-page driver: `__cdGodTowers(n)` pushes near-invincible towers, `__cdDrive({maxWave, cap})` runs the sim with `update(1/60)` to a wave boundary or game end, auto-picking drafts. Use the `check(...)` helper for assertions; add one whenever you add behavior. Before committing, spawn a subagent to run `npm test`.
- **The game loop runs on `requestAnimationFrame`, so it pauses when the tab is hidden.** Headless testing through preview_eval must drive the simulation manually: call `update(1/60)` in a loop instead of waiting wall-clock time.
- Standard test recipe (via preview_eval):
  1. `beginGame()` (set `gameMode`/`mapKey`/`diffKey` first), give `gold`, push tower objects directly into `towers`
  2. `startWave()`, then `while (!done) update(1/60)`
  3. If `draftOpen`, click a card: `document.getElementById('draftCards').children[0].click()`
  4. Check `preview_console_logs` for errors afterwards
- Synthetic canvas clicks must set real `clientX/clientY` relative to `cv.getBoundingClientRect()` — the click handler recomputes coords from the event.
- **Always clean up test data afterwards**: remove localStorage keys `cd_save`, `cd_meta`, `cd_campaign`, `cd_best_easy/normal/hard`, reset `meta = {chips:0, talents:{}}; loadMeta()`, and call `backToMenu()` so the user starts fresh.

## Architecture (single `<script>` in tower-defense.html)

Rough section order in the file:

- **Audio**: `tone()` (oscillator), `noise()` (filtered white noise buffer), `SFX` object — every tower type has a distinct sound (tesla = lightning crackle, cannon = sub-bass boom, etc.)
- **Maps & paths**: `MAPS` (classic/spiral/serpent/mayhem), `genPathWith(rnd)` generates random axis-aligned serpentine paths, `buildPath()` fills `waypoints/segs/pathLen`, `pointAt(dist)`, `distToPath(x,y)`
- **Modes**: `gameMode` = `'quick'` | `'campaign'`; `campLevel` 1–40, `CAMPAIGN_LEVELS = 40`. Campaign maps are **freshly random each attempt** (deliberate design choice). `victoryWave()` = `14 + campLevel` for campaign (L1=15 … L40=54), 30 for quick. Campaign enemy HP scales `1 + (campLevel-1)*0.04`. Progress = localStorage `cd_campaign` (highest completed level). Mayhem (quick-mode map) shifts the path + relocates towers every 5 waves and rolls per-wave `WAVE_MODS`.
- **Meta / talents**: `TALENTS` (20 talents in CORE + TOWER MASTERY sections, ~2290 chips to max), `meta = {chips, talents, achievements, stats}` in localStorage `cd_meta`. Chips earned per run via `chipsForRun()`. `maxTowerLevel()` = 5 + overdrive rank. `loadMeta()` additively migrates older saves (defaults `achievements`/`stats` when absent — never break this).
- **Achievements** (v1.5.0, 9 badges as of v1.8.0): `ACHIEVEMENTS` array + `ACH_BY_ID`. `meta.achievements` is an id→true map; `meta.stats = {dmg, runs, bestCombo}` are lifetime counters (`bestCombo` added v1.8.0, defaulted to 0 in `loadMeta()`). `grantAchievements(won)` is called once per finished run from `endGame(false)`/`winGame(true)` — it tallies `stats` (including `meta.stats.bestCombo = max(bestCombo, comboBest)` from the run's peak streak), evaluates each badge's condition, and returns the newly-unlocked list, which `achLine()` appends to the overlay text. Per-run `livesLostThisRun` flag (set wherever a life leaks; forced `true` in `loadRun()` so resumed runs can't earn **Flawless**). The **Combo Master** badge (`combo30`, v1.8.0) is granted when the run's `comboBest >= 30`. Start-screen 🏅 button → `openAchievements()`/`renderAchievements()` panel (`#achPanel`, mirrors talents); `#achBtn` shows `done/total` (updated in `renderStartScreen`). Adding a badge = push to `ACHIEVEMENTS` + a condition line in `grantAchievements`.
- **Records / personal bests** (v1.6.0): start-screen 🏆 button → `openBests()`/`renderBests()` panel (`#bestPanel`, mirrors achievements). Shows a `.bestTbl` matrix of highest wave per quick-mode **map × difficulty**, an `★ Any map` row (from the legacy per-diff bests), and a `.bestStats` footer (campaign level, lifetime dmg, **best combo** [v1.8.0], runs, chips). Best-wave is recorded by `recordBest()` (called from `endGame`/`winGame`, replacing the old inline `if (wave > best)` lines): it updates the legacy `cd_best_<diff>` key (via `bestKey()`) **and**, in quick mode only, a new additive `cd_best_<map>_<diff>` key. Campaign runs never write per-map keys (maps are random per attempt). All keys read with `|| 0` fallback — old saves unaffected. Map keys (classic/spiral/serpent/mayhem) can't collide with diff keys (easy/normal/hard). **New-record flourish (v1.6.1):** `recordBest()` returns a `{prev, now}` event when a quick-mode run beats its per-map cell (only when `prev > 0` — first-ever entries record silently and return `null`); `endGame()`/`winGame()` pass it to `applyRecordFlourish(rec)`, which shows the golden `#ovRecord` banner (toggled by an `.record` class on `#overlay`, CSS pop/pulse animations), plays `SFX.record()` (delayed ~450ms), and adds shake + a particle burst. `applyRecordFlourish(null)` clears the banner, so every overlay show resets it (no stale banner). DOM/audio only — no save impact; campaign never fires it.
- **Version / What's New**: `GAME_VERSION` constant (semver-ish, bumped every routine run) + `CHANGELOG_ENTRIES` array (last ~10, newest first, mirrors CHANGELOG.md) near the top of the script. **Layout (as of v1.4.1):** the whole game lives in a `#gameCol` column (h1, `#hud`, `#stage`→`#gameWrap`/canvas, `#shop`, `#controls`, `.hint`), and the `#whatsnew` `<aside>` is its **sibling** inside an outer `#appRow` flex row (`align-items:flex-start; flex-wrap:wrap`). So the panel floats beside the **entire game** — opening it shifts the whole column together (not just the canvas, which was the v1.4.0 complaint). The panel is **open by default**, top-aligned with the game column, flush against its right edge (`#appRow` gap 0, `border-left:none`, `border-radius:0 8px 8px 0`, `align-self:stretch`), **height-capped to the game** (v1.5.2): a `syncWhatsNewHeight()` helper sets the panel's inline `max-height` to `#gameCol.offsetHeight` (called from `openWhatsNew()` and on `window` `resize`), so on tall viewports it never overhangs the bottom of the board — overflowing entries scroll internally via `#wnList` instead. (The static CSS `max-height:88vh` stays as a no-JS fallback but the inline cap wins; the row is `align-items:flex-start`, so `#gameCol`'s height is independent of the panel and there's no feedback loop.) It wraps below `#gameCol` on narrow viewports. `initWhatsNew()` (called at startup) opens it unless the player closed it before — `closeWhatsNew()` persists `cd_wnclosed='1'`, `openWhatsNew()` clears it. The start screen has a `✨ What's New` button + clickable `#verTag` (version, set in `renderStartScreen`) to reopen, and the panel has a `✕` (`#wnClose`) to hide it. When you ship a run, bump `GAME_VERSION` and prepend a `CHANGELOG_ENTRIES` entry.
- **Towers**: `TOWER_TYPES` (gun/sniper/frost/cannon/tesla/poison/buff), `SPECS` (2 specializations per type, chosen free at level 5+), targeting `MODES` (first/last/strong/close)
- **Abilities**: `ABILITIES` (meteor Q / freeze W / gold rush E), `abilityCd`, meteor is click-to-target via `armedAbility`
- **Run perks**: `PERKS` with rarity tiers (common/rare/legendary), drafted 1-of-3 every 5 waves (`openDraft`, weighted by `rollRarity()` ≈ 78/14/8 common/rare/legendary). Legendary = "super grades" (Diamond Core, Midas, Orbital Support, Ascension…). `perkState` (from `freshPerkState()`) holds all run-modifier numbers; `REPEATABLE` perks can be offered again.
- **Effective stats**: `effDmg/effRate/effRange/effBuffPower/effBuffRange` combine base stats × talents × perks × specs × buffs × wave mods. **Booster auras do NOT stack — `buffMultFor` takes the max** (deliberate; UI says so).
- **Save/resume**: `saveRun()` after each wave + on quit (mid-wave quit saves `wave-1`); `loadRun()` rebuilds tower stats from base × level multipliers (dmg ×1.45, range ×1.08, rate ×0.88 per level). Random-path modes (mayhem/campaign) call `relocateTowers()` on resume.
- **Game flow**: `startWave → endWave` (interest, draft every 5 waves, auto-start timer) → `winGame`/`endGame` (chips awarded, MVP/perk lines) → `continueEndless`/`nextLevel`/`backToMenu`. `quitRun()` needs two clicks within 3s.
- **Idle chrome (v1.5.1)**: when no game is active, the in-game chrome (stats `#hud`, tower `#shop`, wave `#controls`, hotkey `.hint`) is dimmed (opacity .3 + grayscale + `pointer-events:none`) via an `.idle` class on `#gameCol`. `setActiveUI()` toggles it from the `started` flag and is called in `beginGame`/`loadRun` (lit), `backToMenu` (dimmed), and at startup. Chrome stays lit during the game-over overlay (still an active session); re-dims only on return to menu. Pure CSS + one helper, no save impact.
- **Kill-streak combo (v1.7.0)**: consecutive kills within `COMBO_WINDOW` (2s) build a streak. Run-only state `comboCount/comboTimer/comboBest/comboFlash` (declared by the globals, **reset in `resetState()`**, never saved). `damage()` bumps `comboCount`/refreshes `comboTimer`/sets `comboFlash=1` on every death and, at milestones (`=== 5 || (>=10 && %10===0)`), fires `SFX.combo(n)` (rising chirp, pitch by tier) + shake + an `addExplosion` burst + a `🔥 N× COMBO!` floater. `update()` decays `comboTimer` (→ resets `comboCount` to 0 at 0) and `comboFlash`. `draw()` renders a **top-left** meter (`× N` + "COMBO" + draining timer bar; top-left chosen to clear the top-right `#abilityBar` overlay and the centered boss bar — v1.7.1) when `comboCount>=2 && comboTimer>0`, color from `comboColor(n)` (green→gold→orange→red→purple at 10/20/30/50), pulse-scaled by `comboFlash`. **Purely cosmetic in-game — no gold/economy impact.** The combo state vars are run-only and never saved; the only persistent hook is at end-of-run, where `grantAchievements()` reads the run's `comboBest` peak into the lifetime `meta.stats.bestCombo` stat and grants the **Combo Master** badge at `>= 30` (v1.8.0).
- **Rendering**: pseudo-3D canvas — shaded cylinder towers, radial-gradient sphere enemies, ground shadows, path depth bevel, projectile trails, starfield, vignette, boss HP bar, screen `shake`.
- **Upgrade panel**: rebuilds itself when `upgradeKey(t)` changes (gold affordability, level, spec, *effective* stats) — this is how it live-updates when boosters/perks land while open. Keep new stat sources reflected in `upgradeKey`.

## Conventions & gotchas

- localStorage keys all prefixed `cd_`: `cd_save`, `cd_meta`, `cd_campaign`, `cd_best_<diff>`, `cd_best_<map>_<diff>` (per-map quick-mode best, v1.6.0), `cd_mute`, `cd_wnclosed` (What's New panel hidden by player)
- Speed toggle just runs `update()` 1–3× per frame; pause/draft/menu gate at the top of `update()`
- `damage()` handles armor (reduced by piercing talent), kill credit (`src.dealt/kills`), bounty (fortune talent, midas perk, bountyAdd), splitter children via `pendingSpawns`
- Enemy kinds: norm, fast (w3+), tank (w5+), heal (w7+), shield/armored (w9+), split (w11+), boss every 5th wave
- When adding a perk/talent that changes a stat, wire it into the corresponding `eff*()` helper AND `upgradeKey()` if it shows in the panel
- User's stated preferences so far: loves additive/addictive loops, wanted talents expensive/grindy (~50 runs to max), campaign maps random per attempt, booster non-stacking is fine but must be labeled
